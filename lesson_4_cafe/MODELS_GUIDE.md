# Руководство по работе с 3D моделями в Godot

## Какой формат моделей лучше использовать?

### Рекомендуемый формат: **GLB/GLTF** ✅

**GLB/GLTF** - это лучший выбор для Godot, потому что:
- ✅ Поддерживается нативно (без дополнительных плагинов)
- ✅ Сохраняет материалы и текстуры
- ✅ Поддерживает анимации
- ✅ Оптимизирован для веб и игр
- ✅ Отличное качество импорта

### Альтернативные форматы:

**OBJ** - можно использовать, но:
- ⚠️ Требует отдельные файлы для материалов (.mtl)
- ⚠️ Не поддерживает анимации
- ⚠️ Больше проблем с текстурами

**STL** - не рекомендуется:
- ❌ Только геометрия, без материалов
- ❌ Очень большие файлы
- ❌ Плохая поддержка в Godot

## Как импортировать модель в Godot:

### Шаг 1: Подготовка модели

1. Убедитесь, что модель в формате **GLB** или **GLTF**
2. Если у вас OBJ/STL, конвертируйте в GLB:
   - Используйте Blender: File → Export → glTF 2.0 → выберите .glb
   - Или онлайн конвертеры (например, https://products.aspose.app/3d/conversion)

### Шаг 2: Импорт в Godot

1. Перетащите файл модели (.glb) в папку проекта в FileSystem (левый нижний угол)
2. Godot автоматически создаст файл `.import` с настройками
3. Модель будет готова к использованию!

### Шаг 3: Использование модели в сцене

#### Вариант 1: Заменить простую геометрию на модель

**Для монетки:**
1. Откройте сцену `coin.tscn`
2. Выберите MeshInstance3D с CylinderMesh
3. В Inspector → Mesh выберите "Load" или "Quick Load"
4. Найдите вашу модель монетки (например, `coin.glb`)
5. Модель заменит цилиндр

**Для игрока:**
1. Откройте сцену `cafe.tscn`
2. Найдите Player → MeshInstance3D
3. Замените Mesh на вашу модель персонажа

**Для кассира:**
1. В сцене `cafe.tscn` найдите Cashier
2. Замените BoxMesh на модель кассира

**Для еды:**
1. Откройте `food.tscn`
2. Замените BoxMesh на модель еды

#### Вариант 2: Использовать модель как дочерний узел

1. В сцене добавьте новый Node3D
2. Right Click → Change Type → **MeshInstance3D** (или добавьте MeshInstance3D как дочерний)
3. В Inspector → Mesh → Load → выберите вашу модель

#### Вариант 3: Использовать сцену модели

1. Если модель сложная (с анимациями), откройте её двойным кликом
2. Godot покажет структуру модели
3. Сохраните как сцену (если нужно)
4. Используйте Instance Child Scene в вашей основной сцене

## Настройка масштаба модели

Если модель слишком большая или маленькая:

1. Выберите MeshInstance3D с моделью
2. В Inspector найдите **Transform → Scale**
3. Измените значения (например, 0.1, 0.1, 0.1 для уменьшения в 10 раз)
4. Или в Scene → Transform → Scale

## Настройка позиции модели

Если модель стоит не на нужной высоте:

1. Выберите узел с моделью
2. В Inspector → Transform → Position
3. Измените Y (высота)

## Примеры использования:

### Монетка с моделью:
```
coin.tscn (Area3D)
  └─ MeshInstance3D (заменить CylinderMesh на coin.glb)
  └─ CollisionShape3D (можно оставить CylinderShape3D или изменить на модель)
```

### Персонаж с моделью:
```
Player (CharacterBody3D)
  └─ MeshInstance3D (заменить CapsuleMesh на character.glb)
  └─ CameraPivot (для камеры)
  └─ CollisionShape3D (можно оставить CapsuleShape3D)
```

## Советы:

1. **Используйте простые модели** для детского проекта (низкополигональные)
2. **Проверяйте размеры** - модели могут быть в разных единицах измерения
3. **Оптимизируйте текстуры** - если модель тяжелая, уменьшите разрешение текстур
4. **Тестируйте производительность** - слишком много полигонов может замедлить игру

## Где найти бесплатные модели:

- **Sketchfab** - много бесплатных моделей в GLB формате
- **Poly Haven** - качественные бесплатные модели
- **OpenGameArt** - модели для игр
- **Blender** - можно создать простые модели самостоятельно

## Для преподавателя:

Рекомендуется подготовить модели заранее в формате GLB:
- Монетка (coin.glb)
- Персонаж (character.glb или player.glb)
- Кассир (cashier.glb)
- Еда (burger.glb, pizza.glb, drink.glb)

Дети смогут просто заменить простые геометрические фигуры на красивые модели через интерфейс Godot!
